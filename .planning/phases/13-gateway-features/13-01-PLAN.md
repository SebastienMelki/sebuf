```go
type KrakenDConfig struct {
    Schema    string     `json:"$schema"`
    Version   int        `json:"version"`

}
```

**Part A: Add rate limiting resolver and builder functions to generator.go**

Add to `internal/krakendgen/generator.go`:

1. `resolveRateLimit(gwConfig, epConfig)` -- returns `*krakend.RateLimitConfig`. If `epConfig != nil && epConfig.GetRateLimit() != nil`, return method-level. Otherwise return `gwConfig.GetRateLimit()`. Return nil if neither has it.

2. `resolveBackendRateLimit(gwConfig, epConfig)` -- same pattern for `BackendRateLimitConfig`.

3. `buildRateLimitRouterConfig(rl *krakend.RateLimitConfig) map[string]any` -- builds the endpoint-level rate limit config map. Only include fields with non-zero values. Fields: `max_rate`, `capacity`, `every`, `client_max_rate`, `client_capacity`, `strategy`, `key`.

4. `buildBackendRateLimitConfig(brl *krakend.BackendRateLimitConfig) map[string]any` -- builds the backend-level rate limit config map. Fields: `max_rate`, `capacity`, `every`. Only non-zero values.

5. `buildEndpointExtraConfig(gwConfig, epConfig)` -- returns `map[string]any` or nil. Calls `resolveRateLimit` and if non-nil, adds to map under `NamespaceRateLimitRouter`. Returns nil if map is empty (so omitempty works).

6. `buildBackendExtraConfig(gwConfig, epConfig)` -- returns `map[string]any` or nil. Calls `resolveBackendRateLimit` and if non-nil, adds to map under `NamespaceRateLimitProxy`. Returns nil if map is empty.

**Integrate into GenerateService:** After building the `ep` Endpoint struct, call `buildEndpointExtraConfig(gwConfig, epConfig)` and assign to `ep.ExtraConfig`. Call `buildBackendExtraConfig(gwConfig, epConfig)` and assign to `ep.Backend[0].ExtraConfig`.

IMPORTANT: Use the namespace constants from `namespaces.go`, never inline strings.
