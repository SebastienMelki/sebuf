  <action>
Add circuit breaker, caching, and concurrent calls generation to `internal/krakendgen/generator.go`:

**Circuit Breaker (backend-level extra_config):**

1. `resolveCircuitBreaker(gwConfig, epConfig)` -- returns `*krakend.CircuitBreakerConfig`. Method-level overrides service-level entirely. If `epConfig != nil && epConfig.GetCircuitBreaker() != nil`, return method-level. Otherwise `gwConfig.GetCircuitBreaker()`. Return nil if neither.

2. `validateCircuitBreaker(cb *krakend.CircuitBreakerConfig, serviceName string) error` -- if cb is non-nil, verify all three required fields are positive: `interval > 0`, `timeout > 0`, `max_errors > 0`. Return error like `"service %s: circuit_breaker requires interval, timeout, and max_errors (all must be > 0)"`. Call this in GenerateService before building endpoints -- fail fast on invalid config.

3. `buildCircuitBreakerConfig(cb *krakend.CircuitBreakerConfig) map[string]any` -- builds config map with `interval`, `timeout`, `max_errors` (always present since validation ensures they are > 0). Include `name` if non-empty. Include `log_status_change` if true.

**Backend Caching (backend-level extra_config):**

4. `resolveCache(gwConfig, epConfig)` -- returns `*krakend.CacheConfig`. Method-level overrides service-level entirely. Same pattern as other resolvers.

5. `validateCache(cache *krakend.CacheConfig, serviceName string) error` -- if cache is non-nil and either `max_items > 0` or `max_size > 0`, verify both are non-zero. Return error like `"service %s: cache max_items and max_size must both be set or both unset"`. Call in GenerateService before building endpoints.

6. `buildHTTPCacheConfig(cache *krakend.CacheConfig) map[string]any` -- builds config map. Include `shared` ONLY if true. Include `max_items` and `max_size` only if both > 0.

**Concurrent Calls (top-level endpoint field, NOT extra_config):**

7. `resolveConcurrentCalls(gwConfig, epConfig)` -- returns `int32`. If `epConfig != nil && epConfig.GetConcurrentCalls() > 0`, return method-level. Otherwise `gwConfig.GetConcurrentCalls()`. Return 0 if neither (0 = omit via omitempty).

**Integrate into GenerateService:**

- Before the endpoint-building loop, validate circuit breaker and cache configs at the service level: call `validateCircuitBreaker` on `gwConfig.GetCircuitBreaker()` and `validateCache` on `gwConfig.GetCache()`. These are "default" configs that might have issues.

- Inside the endpoint loop, for each method with `epConfig`:
  - If `epConfig` has circuit breaker, validate it too.
  - If `epConfig` has cache, validate it too.
  - Resolve concurrent calls and assign to `ep.ConcurrentCalls` (top-level field, NOT extra_config).

- Update `buildBackendExtraConfig` to also handle circuit breaker and caching:
  - Call `resolveCircuitBreaker(gwConfig, epConfig)`, if non-nil add to map under `NamespaceCircuitBreaker`.
  - Call `resolveCache(gwConfig, epConfig)`, if non-nil add to map under `NamespaceHTTPCache`.

IMPORTANT: Circuit breaker and cache go in BACKEND extra_config. Not endpoint extra_config. Rate limit router and JWT go in ENDPOINT extra_config. This distinction is critical.
IMPORTANT: `concurrent_calls` goes on `ep.ConcurrentCalls` field, NOT in any extra_config.
IMPORTANT: Use namespace constants from namespaces.go.
