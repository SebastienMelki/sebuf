2. Create `examples/krakend-gateway/gateway/krakend.tmpl` -- the Flexible Config template that composes per-service endpoint fragments into a complete krakend.json. Use the partials approach:
   - The template should include the standard $schema and version 3 fields
   - Use `{{ include "user_endpoints.json" }}` and `{{ include "product_endpoints.json" }}` within the endpoints array
   - The partials (created by `make partials` using jq) contain bare JSON arrays of endpoint objects. The template needs to unwrap these arrays and combine them.
   - Simplest working approach: each partial is a `.endpoints` array extracted by jq. The template concatenates them:

```
{
  "$schema": "https://www.krakend.io/schema/krakend.json",
  "version": 3,
  "endpoints": [
    {{ include "user_endpoints.json" }},
    {{ include "product_endpoints.json" }}
  ]
}
```

Where the jq extraction step produces partial files that contain just the array contents (not the array brackets). Use: `jq '.endpoints | .[]' generated/UserService.krakend.json | jq -s '.'` to get a bare array, then the include wraps it.

Actually, the simplest reliable approach: each partial IS a JSON array (e.g., `[{...}, {...}]`). The template does NOT put them inside another array. Instead it uses the settings + range approach to unwrap and combine:

OR simplest: partials are the raw contents of the endpoints arrays as comma-separated objects (no outer brackets). The Makefile jq step strips the brackets:
```bash
jq -r '.endpoints | map(tostring) | join(",\n")' generated/UserService.krakend.json > gateway/partials/user_endpoints.json
```
Wait -- that would stringify each endpoint. Instead:
```bash
# Extract endpoints array content (without brackets) using jq
jq '.endpoints' generated/UserService.krakend.json | sed '1d;$d' > gateway/partials/user_endpoints.json
```

This removes the first `[` and last `]` lines, leaving comma-separated endpoint objects that can be directly included.

Then the template is simply:
```
{
  "$schema": "https://www.krakend.io/schema/krakend.json",
  "version": 3,
  "endpoints": [
    {{ include "user_endpoints.json" }}
    ,
    {{ include "product_endpoints.json" }}
  ]
}
```

Test this by running `make compose` and verifying the output passes `krakend check -lc`.
