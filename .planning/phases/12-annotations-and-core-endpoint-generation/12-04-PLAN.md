Create `internal/krakendgen/validation.go` with route conflict detection.

**Public function:**
```go
func ValidateRoutes(endpoints []Endpoint, serviceName string) error
```

Takes the list of generated endpoints and the service name. Returns an error if any conflicts are detected.

**Validation 1 — Duplicate (path, method) tuples (VALD-01):**
Build a map of `(endpoint_path, method)` -> RPC name. If a key already exists when inserting, return an error like:
```
service UserService: duplicate route: GET /api/v1/users (from GetUsers and ListUsers)
```

**Validation 2 — Static vs parameterized route conflicts (VALD-02):**
Build a trie of path segments, grouped by HTTP method. For each method group:
- Split each endpoint path into segments (split on `/`, skip empty).
- Walk/insert into the trie segment by segment.
- A segment starting with `{` is parameterized; otherwise it's static (literal).
- At each trie node, if inserting a literal when a param child already exists (or vice versa), report a conflict:
```
service UserService: route conflict at /users/: static segment "search" (GetUsersBySearch) conflicts with parameter "{id}" (GetUser)
```

Implement the trie as unexported types:
```go
type routeNode struct {
    children   map[string]*routeNode
    paramChild *routeNode
    paramName  string
    rpcName    string // set at leaf for error reporting
}
```

**Integration into generator:**
In `GenerateService`, after building all endpoints (but before returning), call `ValidateRoutes(endpoints, serviceName)`. If it returns an error, return it from GenerateService. The plugin entry point already reports generator errors to protoc.

Run `make lint-fix` after implementing.
