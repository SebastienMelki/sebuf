  <action>
Create `internal/krakendgen/generator.go` with a single public entry point:

```go
func GenerateService(service *protogen.Service) ([]Endpoint, error)
```

The function must:

1. **Read gateway_config annotation** (service-level):
   - Use `proto.GetExtension(service.Desc.Options(), krakend.E_GatewayConfig)` pattern (same as `annotations/http_config.go` reads `sebuf.http.config`).
   - If no `gateway_config` exists on the service, return an error: `"service %s has no (sebuf.krakend.gateway_config) annotation -- backend host is required"` where `%s` is the service name. This implements CORE-03/CORE-04 (host is required, comes from annotations only).
   - Extract `host` ([]string) and `timeout` (string) as service-level defaults.

2. **Iterate RPCs and build endpoints**:
   - For each method in the service, call `annotations.GetMethodHTTPConfig(method)` to get the HTTP path and method.
   - If the method has no `sebuf.http.config`, skip it silently (per user decision).
   - Get the service base path via `annotations.GetServiceBasePath(service)`.
   - Build the full path via `annotations.BuildHTTPPath(basePath, httpConfig.Path)`.
   - Use `httpConfig.Method` directly -- `GetMethodHTTPConfig` already returns the uppercase method string (e.g., "GET", "POST"). No conversion needed.

3. **Read endpoint_config annotation** (method-level):
   - Use `proto.GetExtension(method.Desc.Options(), krakend.E_EndpointConfig)` pattern.
   - If present, override service-level host and timeout (ANNO-04: method overrides service).

4. **Assemble Endpoint struct**:
   - `Endpoint`: the full HTTP path (passed through as-is, `{param}` syntax maps directly to KrakenD)
   - `Method`: always set explicitly (never rely on KrakenD's GET default) -- use uppercase HTTP method
   - `OutputEncoding`: always `"json"` (CORE-06)
   - `Timeout`: set only if annotated at service or method level; omit if not annotated (per user decision). Method-level overrides service-level.
   - `InputHeaders`: leave nil for now (Plan 03 will populate)
   - `InputQueryStrings`: leave nil for now (Plan 03 will populate)
   - `Backend`: single Backend entry with:
     - `URLPattern`: same as endpoint path (passthrough, no remapping)
     - `Host`: from gateway_config or endpoint_config override
     - `Method`: same as endpoint method
     - `Encoding`: `"json"`

5. **Return the endpoints slice** (order follows RPC declaration order).

Helper functions to create (unexported):
- `getGatewayConfig(service *protogen.Service) (*krakend.GatewayConfig, error)` -- reads service extension, returns error if missing
- `getEndpointConfig(method *protogen.Method) *krakend.EndpointConfig` -- reads method extension, returns nil if absent
- `resolveHost(gwConfig *krakend.GatewayConfig, epConfig *krakend.EndpointConfig) []string` -- returns method-level host if non-empty, else service-level
- `resolveTimeout(gwConfig *krakend.GatewayConfig, epConfig *krakend.EndpointConfig) string` -- returns method-level timeout if non-empty, else service-level

IMPORTANT per research: Use `descriptorpb.ServiceOptions` / `descriptorpb.MethodOptions` type assertions before `proto.GetExtension`, matching the exact pattern in `internal/annotations/http_config.go`.
